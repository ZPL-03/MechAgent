# MechAgent 长周期开发计划书

> 基线版本：`0.1.0`（结构线弹性静力闭环）。计划基线时间：2026 年 6 月。
> 本文是面向开源 CAE 智能体平台的长周期路线，定义从结构静力基线走向通用、可信、可扩展 CAE/FEA 多智能体系统的目标、阶段、验收与治理。产品定位与当前能力见 [产品蓝图](product_blueprint.md)，实现细节见 [架构与数据契约](architecture.md)。

## 1. 愿景与定位

MechAgent 的目标不是再实现一个有限元求解器，也不是只覆盖梁、板、开孔板等固定算例的脚本集合，而是构建一个**通用、开放、可信的 CAE/FEA 多智能体编排平台**：把自然语言工程意图转化为可执行、可审计、可复现、可验证的仿真证据，并随能力注册与工具注册持续扩展到更多物理域、几何族、求解器、网格器和工程工作流。

平台的长期价值主张：

- **意图到证据**：用户用自然语言描述工程问题，系统自动完成几何/参数建模、网格、求解、后处理、校核与报告，并给出可信度与复核建议。
- **能力可扩展**：新物理域、新几何、新求解器、新材料、新评价器通过 `SimulationCapability` 与工具注册进入同一框架，而非硬编码分支。
- **证据可验证**：每个发布能力都有解析参考、工程基准、网格收敛、跨工具对比或实验数据之一作为验收证据。
- **过程可复现**：请求、配置、工具版本、网格、求解输入与输出、报告形成可追溯链路。
- **工程可协作**：从单机工作台扩展到批量、远程/HPC 执行、结果归档与审计追踪。

## 2. 设计原则与不变量

无论扩展到何种物理域，以下不变量在全周期保持：

1. **能力驱动**：业务逻辑围绕 `SimulationCapability` 注册，不围绕测试编号或固定算例编码分支。
2. **结构化通信**：Agent 间使用 Pydantic v2 数据契约，不依赖非结构化文本拼接。
3. **有限数值与执行契约**：进入工具层前统一校验几何、材料、载荷、边界、网格、求解选项；拒绝非有限数值与不一致输入。
4. **验证优先**：能力进入发布边界前必须具备可自动化复核的验收证据；缺参考解时使用工程基准/网格收敛/跨工具对比并在文档说明依据。
5. **脱敏与安全**：公开摘要不输出原始 prompt/response、密钥与本机私密路径；私密配置经 `.env` 注入。
6. **可复现**：求解产物按任务隔离，复现命令与工作台链接可固化到终端与 CI。
7. **文档/代码/测试一致**：发布内容以代码、配置、脚本和测试证据为准。
8. **渐进发布**：先形成可信闭环，再扩展物理域与工具生态；每阶段有明确门禁。

## 3. 现状基线（0.1）

已发布并验证：

- 结构线弹性静力分析：梁、矩形薄板、单孔/偏心孔/多孔/槽孔开孔薄板、矩形实体块。
- 工具：CalculiX 求解、Gmsh 网格（含 OCC 布尔建模）、本地结构化网格与质量门禁。
- 编排：Planner→Designer→MeshAgent→SolverAgent→PostProcAgent→AnalystAgent→ReporterAgent，LangGraph DAG 与顺序工作流共用 Agent 与数据契约。
- 入口：Studio 工作台（明暗双主题、引导式工作流、3D 视口）、CLI、Python SDK。
- LLM：可选 OpenAI 兼容接口参与能力选择、参数补全、网格策略建议与工程解释，脱敏 trace。
- 验证：TC-01~05 真实 CalculiX 验证、二十八个自然语言案例、LLM smoke、质量门禁。

基线的局限即本计划要突破的边界：物理域单一（线弹性静力）、几何受限（参数化简单件）、求解/网格器单一、无 CAD 导入、无优化与平台化能力。

## 4. 目标架构演进

在保持现有分层（交互层 / 编排层 / 能力层 / 工具层 / 数据层）的前提下，按以下方向演进。

### 4.1 工具抽象层扩展

把当前 `AbstractSolver`/`AbstractMesher`/`PostProcessor` 扩展为完整的可插拔工具族，统一注册、工厂创建与配置校验：

| 抽象 | 职责 | 候选实现 |
| --- | --- | --- |
| `SolverAdapter` | 生成求解输入、执行、提取结果 | CalculiX、Code_Aster、Elmer、CalculiX-CGX、FEniCSx、MFEM、Kratos、OpenFOAM、SU2 |
| `MesherAdapter` | 几何离散、质量指标、局部尺寸场 | Gmsh、Netgen、自适应重网格 |
| `CADKernel` | 形体导入、修复、特征识别、简化 | OpenCASCADE（pythonocc）、FreeCAD |
| `PostProcessor` | 场量解析、派生量、切片/流线/动画数据 | meshio、VTK、PyVista |
| `MaterialModel` | 各向同性/各向异性/非线性本构 | 内置目录、外部材料库、用户本构 |
| `Evaluator` | 主结果、参考、误差、验收 | 解析、基准、跨工具、网格收敛 |
| `JobExecutor` | 本地/远程/HPC/容器执行 | 本地进程、Slurm、容器、远程工作站 |
| `KnowledgeSource` / `LLMBackend` | 工程知识检索 / 结构化抽取与解释 | 本地 JSONL、OpenAI 兼容接口 |

### 4.2 数据契约扩展

`ModelParams` 与求解/后处理摘要扩展以支持：非线性分析选项（几何/材料/接触）、瞬态时间步与求解控制、多物理场耦合定义、多载荷工况与载荷步、装配体与接触对、参数扫描与设计变量、网格自适应与收敛控制。扩展保持向后兼容与单点定义。

### 4.3 智能体扩展

在七个核心 Agent 之上，按需引入：

- **GeometryAgent**：CAD 导入、几何修复/简化、特征识别、边界/载荷区域映射。
- **ConvergenceAgent**：网格收敛、非线性/瞬态收敛诊断与自适应重试。
- **OptimizationAgent**：参数扫描、DOE、尺寸/形状/拓扑优化的实验编排。
- **MaterialAgent**：材料识别、本构选择与参数标定。
- **ValidationAgent**：选择验收策略（解析/基准/跨工具/实验），生成验证报告。

新增 Agent 复用同一状态契约、错误记录与脱敏规则。

### 4.4 复现与溯源层

引入 provenance：记录请求、配置快照、工具与版本、网格指纹、求解输入/输出哈希、报告版本，支撑结果归档、审计与可复现实验。

## 5. 能力成熟度模型与发布门禁

每个能力按成熟度推进，进入更高等级需满足对应证据：

| 等级 | 定义 | 进入条件 |
| --- | --- | --- |
| 研究能力 | 识别工程意图，形成候选参数/网格/工具方案 | schema 草案、失败诊断、示例请求 |
| 可执行能力 | 从请求生成求解输入并完成真实求解与后处理 | 端到端求解记录、阶段产物、报告 |
| 可验证能力 | 具备解析参考/工程基准/网格收敛/跨工具/实验证据 | 误差阈值、自动化测试、质量门禁 |
| 发布能力 | Studio/CLI/SDK/文档/清理/安装一致 | 前端构建、包构建、文档构建、完整门禁 |

**发布门禁（每个发布能力）**：可执行求解链路；至少一类验收证据；3D/后处理可视化映射；Markdown 报告字段；自然语言案例；CLI/SDK/Studio 一致；`ruff`/`mypy`/`pytest`/前端构建/MkDocs 构建/清理通过。

## 6. 能力与物理域路线图

按「先形成可信闭环，再扩展物理域与工具生态」推进。每阶段列出目标、关键能力、接入工具、验收证据与风险。阶段可并行推进其中的可验证子能力。

### P1 · 结构线弹性静力（已发布，持续巩固）

- 目标：稳定的结构静力闭环作为回归基线。
- 已覆盖：梁、矩形板、圆孔/多孔/槽孔开孔薄板、实体块；CalculiX + Gmsh；TC-01~05、二十八个自然语言案例。
- 持续项：更多边界/载荷区域、误差阈值收紧、自然语言鲁棒性、报告与可视化打磨。

### P2 · 复杂几何前处理与几何泛化

- 目标：从参数化简单件走向真实工程几何。
- 关键能力：CAD 导入（STEP/IGES/BREP）、几何修复与去特征（defeaturing）、特征识别、参数化零件库、二维支架/加强筋/圆角/倒角轮廓、三维支架/壳体、简单装配体、局部网格尺寸场、网格自适应与网格收敛、多网格器（Netgen）。
- 接入工具：OpenCASCADE/pythonocc、FreeCAD、Netgen、pygmsh。
- 验收证据：CAD 样例、几何修复失败诊断、网格质量指标、网格收敛曲线、真实求解结果与 UI 多任务验收。
- 风险：CAD 质量参差、修复鲁棒性、网格生成失败率；以诊断与回退策略缓解。

### P3 · 高级结构分析

- 目标：覆盖工程常见的非线性与特征值类分析。
- 关键能力：几何非线性（大变形/大转动）、材料非线性（弹塑性、超弹性、蠕变）、接触（摩擦/无摩擦）、线性与非线性屈曲、模态/特征值、谐响应/频响、基础疲劳与渐进损伤。
- 接入工具：CalculiX 非线性能力、Code_Aster（高级非线性/接触）、Elmer。
- 验收证据：解析参考（屈曲临界载荷、固有频率）、工程基准、跨工具对比、非线性/特征值收敛诊断、报告字段。
- 风险：非线性收敛与求解器差异；以 ConvergenceAgent 与跨工具校验缓解。

### P4 · 动力学与瞬态过程

- 目标：时间相关结构响应。
- 关键能力：隐式/显式瞬态动力学、冲击/碰撞、时间积分与阻尼、响应谱、随机振动。
- 接入工具：CalculiX、Code_Aster、Kratos。
- 验收证据：解析/半解析参考（单自由度、梁动力学）、守恒量检查、时程后处理与动画可视化。
- 风险：时间步与稳定性、产物体量；以自适应步长与结果抽样缓解。

### P5 · 多物理场与场耦合

- 目标：热、电磁、声学及其与结构的耦合。
- 关键能力：稳态/瞬态热分析、热-结构耦合（热应力）、电磁（静/低频）、声学、压电；单向与双向耦合。
- 接入工具：Elmer（多物理）、CalculiX（热-结构）、OpenFOAM（传热）。
- 验收证据：多场量后处理、守恒量检查、解析/基准参考、端到端耦合案例。
- 风险：耦合迭代与收敛、场量映射；以标准化场映射与守恒检查缓解。

### P6 · 流体与流固耦合（CFD/FSI）

- 目标：把平台从固体力学扩展到流体与多体耦合。
- 关键能力：不可压/可压流动、湍流模型、共轭传热、流固耦合（单向/双向）。
- 接入工具：OpenFOAM、SU2、preCICE（耦合中间件）。
- 验收证据：经典 CFD 基准（顶盖驱动腔、圆柱绕流、平板边界层）、网格收敛、跨工具对比。
- 风险：CFD 计算成本与稳定性、网格质量要求高；以分级算例、远程执行与代理模型缓解。

### P7 · 优化与设计探索

- 目标：从单次仿真走向设计闭环。
- 关键能力：参数扫描、DOE、灵敏度分析、尺寸/形状/拓扑优化、代理模型（响应面/高斯过程）、不确定性量化（UQ）、鲁棒/可靠性优化。
- 接入工具：OpenMDAO、Dakota、pymoo、scipy.optimize、scikit-learn（代理）。
- 验收证据：优化收敛历史、与解析最优对比、代理模型精度、可复现实验记录。
- 风险：仿真次数与成本；以代理模型、并行执行与早停缓解。

### P8 · 平台化与工程协作

- 目标：从单机工作台走向可运维、可协作的平台。
- 关键能力：作业队列、分布式/远程/HPC 执行（Slurm/容器）、结果归档与检索、审计与溯源、能力插件 SDK 与打包、公开 API 与文档站、协作与权限、基准数据集。
- 接入工具：容器运行时、Slurm、对象存储、任务队列。
- 验收证据：队列任务验收、产物索引与溯源、wheel/插件安装验证、端到端 UI/平台验收。
- 风险：分布式复杂度与安全；以最小可用执行器、隔离与审计缓解。

## 7. 工具生态接入矩阵

平台以开放工具生态为核心。候选开源工具按域登记，每个工具以注册名、工厂函数、配置与执行契约接入，并提供能力级验收证据。

| 域 | 候选工具 | 接入形态 |
| --- | --- | --- |
| 结构求解 | CalculiX、Code_Aster、Elmer、FEniCSx、MFEM、Kratos | `SolverAdapter` |
| 流体求解 | OpenFOAM、SU2 | `SolverAdapter` |
| 耦合中间件 | preCICE | 耦合执行器 |
| 网格 | Gmsh、Netgen、pygmsh | `MesherAdapter` |
| CAD 内核 | OpenCASCADE/pythonocc、FreeCAD | `CADKernel` |
| 后处理 | meshio、VTK、PyVista、ParaView | `PostProcessor` |
| 材料 | 内置目录、外部材料库、用户本构 | `MaterialModel` |
| 优化 | OpenMDAO、Dakota、pymoo、scipy | `OptimizationAgent` 调度 |
| 代理/UQ | scikit-learn、高斯过程、Chaospy | 代理注册 |
| 执行 | 本地、Slurm、容器、远程工作站 | `JobExecutor` |

接入任何工具的最低交付边界：可复现求解、可解释报告、至少一类验收证据、Studio 可视化映射、文档说明。许可证与依赖以各工具上游为准，不绑定特定商业求解器。

## 8. 智能体与 LLM 演进

- **能力图规划**：Planner 在能力注册表上做结构化能力选择与复合请求拆分；扩展到能力依赖与多步工作流编排。
- **结构化工具使用**：Designer/MeshAgent/SolverAgent 的 LLM 输出经 schema、执行契约与工具适配器约束后进入链路，本地解析优先。
- **自校正与收敛回路**：ConvergenceAgent 基于网格收敛、非线性/瞬态收敛诊断驱动自适应重试，纳入 trace 审计。
- **检索增强**：知识库扩展工程规则、材料、求解器配置与最佳实践，参与能力选择、网格策略与报告。
- **评测体系（eval harness）**：建立 LLM 链路评测集，度量能力识别准确率、参数抽取正确率、缺参诊断与报告质量，纳入回归。
- **护栏与脱敏**：保持原始 prompt/response/密钥不入公开摘要；远端失败短超时回退，主链路按确定性数据继续。

## 9. 质量、验证与基准体系

- **分级门禁**：便携 CI（`check_env.py --profile portable`、`pytest -m "not real_solver"`、前端构建）与本地完整门禁（真实求解、自然语言案例、LLM smoke、包/文档构建）。
- **域基准套件**：每个物理域建立标准算例与参考（解析/基准/跨工具/实验），自动比对误差与阈值。
- **网格收敛自动化**：对支持的能力自动生成收敛序列与曲线作为验收证据。
- **跨工具一致性**：同一问题跨求解器/网格器对比，记录差异与原因。
- **回归与黄金数据**：固化关键算例的黄金结果，防止数值漂移；失败优先归一化保持验收语义。

## 10. 可视化与报告演进

- **多场量后处理**：位移、应力、应变、温度、流场、模态振型、屈曲模态等场量统一颜色映射、图例与单位。
- **瞬态与动画**：时程结果抽样、动画、切片、等值面、流线、矢量场。
- **对比与历史**：多任务/多方案对比视图、优化与收敛历史曲线。
- **报告模板**：按分析类型提供报告模板，覆盖模型、网格、求解、结果、验收、复核建议与执行链路摘要；确定性解读为基线，LLM 工程解释为增强。

## 11. 平台、运维与协作

- **作业模型**：在现有 Studio 后端作业模型上扩展队列、优先级、取消与重试。
- **执行器**：本地 → 远程工作站 → HPC（Slurm）→ 容器化，统一 `JobExecutor` 契约。
- **归档与溯源**：结果目录、产物索引、provenance 记录、可复现实验记录与审计追踪。
- **插件与 API**：能力插件 SDK 与打包规范、公开 API 文档、文档站点与示例库。
- **安全**：私密配置 `.env` 注入、密钥脱敏、外部程序路径由配置与完整可执行路径管理、远程执行隔离。

| 里程碑 | 目标 | 主要交付 | 验收证据 | 估计窗口 |

| 0.1（已发布） | 结构静力闭环 | 梁/板/圆孔、多孔与槽孔开孔板/实体块、Studio/CLI/SDK | TC-01~05、28 案例、LLM smoke | 已完成 |
| 0.2 | 复杂几何前处理（P2） | CAD 导入、几何修复、特征识别、自适应网格、多网格器 | CAD/参数案例、网格收敛、UI 多任务 | 2026 H2 – 2027 Q1 |

## 13. 风险与缓解

| 风险 | 影响 | 缓解 |
| --- | --- | --- |
| 求解器兼容/许可差异 | 接入受阻、结果不一致 | 开放工具优先、跨工具对比、执行契约隔离差异 |
| 数值鲁棒性（非线性/CFD 收敛） | 失败率高、结果不可信 | ConvergenceAgent、自适应重试、收敛诊断与门禁 |
| LLM 可靠性与幻觉 | 参数/能力误判 | 本地解析优先、schema/契约约束、eval harness、脱敏回退 |
| 验证负担随物理域增长 | 发布变慢 | 分级门禁、域基准套件、网格收敛自动化、便携 CI |
| 计算成本与规模 | 体验与成本压力 | 代理模型、并行/远程执行、结果抽样与缓存 |
| 范围蔓延 | 资源分散 | 成熟度模型与发布门禁、按里程碑收敛、可验证子能力优先 |
| 安全与私密泄露 | 凭证/路径外泄 | `.env` 注入、脱敏规则、配置化外部路径、远程隔离 |

## 14. 治理与社区

- **贡献模型**：能力以 `SimulationCapability` 插件形式贡献，附解析器、执行契约、评价器、自然语言案例与验收证据；提供能力插件 SDK 与模板。
- **RFC 流程**：跨层变更（新物理域、新工具抽象、契约扩展）走轻量 RFC，明确目标、契约、验收与回退。
- **发布节奏**：按里程碑发布，保持文档/代码/测试一致与可复现安装验证。
- **文档与示例**：每个能力提供使用指南、示例库、基准与报告样例；文档站随发布构建。
- **质量基线**：`ruff`/`mypy`/`pytest`/前端构建/MkDocs 构建/清理脚本作为持续基线。

## 15. 近期可执行子项（0.2 启动）

为把长周期目标落到可执行的近期工作：

1. 抽象 `CADKernel` 与 `JobExecutor` 接口（已落地草案：`AbstractCADKernel`、`AbstractJobExecutor` 与 `LocalCommandExecutor`，含注册/工厂/配置与单元测试），后续接入真实 CAD 内核与远程/HPC 执行后端。
2. CAD 导入与几何信息提取（已落地：基于 gmsh OpenCASCADE 后端的 `GmshCADKernel` 导入 STEP/IGES/BREP，`geometry_candidate_from_summary()` 产出几何候选与边界/载荷区域候选，`GeometryAgent` 串联两者并含单元测试），后续接入几何修复/特征识别与更多 CAD 内核（pythonocc/FreeCAD）。
3. `GeometryCandidate` 到求解链路的对接（已落地：`parse_static_model_params_from_geometry()` 与 `GeometryAgent.to_model_params()` 把几何候选与自然语言补全的材料/载荷/边界合并为 `ModelParams`，经同一网格、求解、后处理与校核链路打通「CAD 文件 → 求解」闭环，含真实 CalculiX 端到端测试），后续由特征识别与载荷语义协同细化几何类型推断。
4. 接入 Netgen 作为第二网格器，建立网格收敛自动化脚手架与质量指标对比。
5. 扩展 `ModelParams` 的非线性/多工况选项（向后兼容），并补充执行契约与缺参诊断。
6. 建立域基准套件目录结构与 `ValidationAgent` 验收策略选择骨架。
7. 扩展 Studio 可视化：多场量切换、网格收敛曲线、多任务对比视图。
8. 建立 LLM eval harness 的最小评测集与回归入口。

以上子项均以可复现求解、可解释报告与至少一类验收证据为最低交付边界。
